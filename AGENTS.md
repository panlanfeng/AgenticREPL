# srun - Smart Run

Type `srun` in your terminal to enter the smart REPL.

## Core Concept

Normal commands execute normally. LLM intervenes only when there's a problem. Zero extra latency.

## Usage

```
$ srun
> ls -la              # normal shell, direct execution, 0 delay
> cat data.csv        # normal shell, direct execution, 0 delay
> cat xx.csv sort by student name | grep scores > 80  # bad command, LLM fixes then runs
> ll                  # alias auto-expanded → ls -la
> 100/4               # arithmetic expression → 25.0
```

## Platforms

macOS, Linux

## Key Principles

- User-entered commands run directly with no safety checks
- Normal commands have zero perceivable delay (<10ms)
- LLM is called only on errors or ambiguous input
- LLM prompt is kept very short; output is executable code only, no explanations
- LLM can use tools to get context (file metadata, system info)
- Default LLM: DeepSeek v4 Flash (OpenAI-compatible API)
- Shell commands generated by LLM go through danger detection; Python code runs directly
- State file: `~/.srun/state.json` (variable schemas, file list, system info)

## API Key

Read from `DEEPSEEK_API_KEY` environment variable. Not hardcoded. To set up for testing:
```bash
export DEEPSEEK_API_KEY="sk-xxx"
```
srun auto-loads env vars from `~/.zshrc` on startup.

## Architecture

```
User Input → Classify (Python / Shell / Unknown)
  ├─ Python AST? → auto-quote column names (region → "region") → exec()
  ├─ Shell?       → subprocess
  └─ Unknown       → LLM dispatch → generate code → execute
                        │
                      Fail → LLM repair (max 4 rounds) → retry with new error
```

## Design

### Classifier (`dispatch.py`)
Determines input type with zero latency:
- **Pseudocode detection first**: "sort by", "filter by", "groupby" → route to LLM
- **Shell detection**: known commands (ls, cat, grep...) or shell operators (|, >, &&)
- **Python detection**: valid AST with imports, calls, attributes, assignments, expressions

### Execution (`executors/`)
- **Python**: in-process `exec()` with AST-based column name resolution (bare `region` → `"region"`)
- **Shell**: `subprocess.run()` with BSD/GNU-aware environment; `cd` persists via CWD sync
- **R**: rpy2 bridge (optional dependency)

### Error Repair Loop (`repl.py`)
```
1. Execute original command
2. If failure or stderr errors detected → call LLM to generate fix
3. Execute fix → if still fails → feed new error back to LLM
4. Repeat up to 4 rounds total
5. Quick fixes (aliases, common typos) bypass LLM entirely (<1ms)
```

### Context (`context.py`)
LLM receives:
- Platform: macOS version, architecture, CPU cores, shell type
- Tools note: BSD vs GNU grep/sed/awk distinction
- Workspace: current directory and file list with sizes
- Session: active DataFrame schemas, variable metadata, last language used

### Danger Detection (`danger.py`)
Regex patterns for destructive shell patterns (rm -rf /, mkfs, dd to /dev, fork bombs, piped curl/wget to shell). Only applies to LLM-generated code, not user input.

## UX Features

- **Readline**: arrow keys, history (~/.srun/history), Ctrl-A/E for line editing
- **Colored output**: CLICOLOR enabled, ls shows directories/files in color
- **cwd in prompt**: `srun:/path/to/cwd$` shows current directory
- **LLM fix preview**: generated/fixed commands shown with `→` prefix

## Tests

### Test Philosophy

**Don't assume tests pass just because they ran.** Verify each output against expected results. A failure is a failure — diagnose root cause first, then decide whether to fix code, fix test, or document as known limitation. Whether the issue is srun code, LLM quality, or environment, it must be investigated.

### Running Tests

```bash
# All tests
pytest tests/ -v

# Fast tests only (no LLM calls)
pytest tests/ -v -m "not slow and not llm"

# LLM tests only
pytest tests/ -v -m "llm"

# Simulated user sessions
pytest tests/test_synthetic_project.py -v -s
```

### Test Checklist

Each test run must verify:
- [ ] Normal shell commands execute within 20ms (zero perceivable delay)
- [ ] Quick fixes (aliases, simple rules) complete within 20ms
- [ ] Pseudocode/natural language correctly translated by LLM
- [ ] Syntax errors/typos fixed by LLM and re-executed successfully
- [ ] Dangerous commands properly blocked
- [ ] `cd` command changes directory persistently
- [ ] `ll`/`la` aliases work correctly
- [ ] srun runs from any directory

### Known Limitations

- LLM fix quality is model-dependent; may not be aware of platform differences (e.g., macOS BSD vs GNU command options)
